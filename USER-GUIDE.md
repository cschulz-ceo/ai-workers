# AI Workers User Guide

**Welcome to your AI-powered team!** This guide helps you understand and work with your autonomous AI agents.

## What is AI Workers?

AI Workers is like having a team of specialized experts working for you 24/7. These AI agents handle tasks like coding, writing, planning, and research - all running on your local computer for complete privacy.

Think of it as having:
- **Kevin** - Your systems architect who designs how things work
- **Jason** - Your software engineer who builds and fixes code
- **Scaachi** - Your marketing expert who creates content and copy
- **Christian** - Your rapid prototyper who quickly builds and tests ideas
- **Chidi** - Your researcher who analyzes feasibility and finds answers

## How to Talk to Your AI Team

### Using Slack (Recommended)

**In any Slack channel:**
```
/ai [agent_name]: your request here
```

**Examples:**
- `/ai kevin: Design a microservices architecture for our e-commerce platform`
- `/ai jason: Write a Python function to process CSV files`
- `/ai scaachi: Write marketing copy for our new AI tool launch`
- `/ai christian: Create a wireframe for a mobile app dashboard`
- `/ai chidi: Research if we can build real-time collaboration with WebSockets`

**In dedicated agent channels:**
- `#counsel-kevin` - Architecture discussions
- `#tasks-jason` - Code development tasks
- `#gen-blueprint` - Architecture outputs
- `#gen-forge` - Code outputs
- `#gen-quill` - Content outputs

### Using Web Interface
Visit `http://localhost:8080` for a chat interface with all agents.

## What to Expect

### Response Times
- **Simple questions**: 15-30 seconds
- **Complex tasks**: 30-60 seconds
- **Large code/projects**: 1-2 minutes

### What Agents Can Do
✅ **Great at:**
- Writing code and scripts
- Creating documentation and plans
- Analyzing problems and solutions
- Generating content and copy
- Designing architectures and systems

❌ **Cannot:**
- Access external websites or APIs
- Execute code directly on your system
- Access personal files or data
- Make real system changes

### When Agents Might Struggle
- Very vague or unclear requests
- Tasks requiring real-time data
- Requests to access private information
- Extremely complex multi-step projects

## Tips for Better Results

### Be Specific
Instead of: "Help with our website"
Try: "Write HTML and CSS for a landing page with hero section, features, and contact form"

### Provide Context
Instead of: "Fix this error"
Try: "I'm getting a 'null pointer exception' in my Python code when processing user data. Here's the error message..."

### Set Clear Expectations
Instead of: "Make it better"
Try: "Optimize this code for performance and add error handling"

### Use the Right Agent
- **Kevin** for architecture and planning
- **Jason** for coding and technical implementation
- **Scaachi** for content and marketing
- **Christian** for prototypes and design
- **Chidi** for research and analysis

## Common Questions

### Q: Can agents work together?
A: Yes! Use `/ai kevin: Design a system` followed by `/ai jason: Implement Kevin's design` - agents can reference each other's work.

### Q: Are my conversations private?
A: Yes! Everything runs locally on your computer. No data leaves your system.

### Q: Can agents access my files?
A: No, agents cannot access your personal files for security. They work with what you provide in the conversation.

### Q: What if I don't know which agent to ask?
A: Start with `/ai kevin:` - he's great at directing questions to the right team member.

### Q: Can agents learn from our conversations?
A: Agents remember context within a conversation, but don't retain information between different conversations.

### Q: How do I get help if something isn't working?
A: Check the [Troubleshooting Guide](TROUBLESHOOTING.md) or use `/ai-diagnose` in Slack for system health checks.
